# Bug Workflow Runbook

> canonical §3 的操作細節。

| 判斷 | T1 編修 | T2 快線 | T3+ 慢線 |
|---|---|---|---|
| 條件 | 已知 typo／文案；無行為影響 | 根因已知、局部、可逆、無紅線 | 根因不明、跨檔、契約／資料／安全影響或紅線 |
| 留痕 | commit 說明 | `BUGS.md` 一行 + trailers | bug 卡 + Log |
| 流程 | 聚焦自查 → commit | 分支 → 先紅後綠 → 獨立輕量查核 → merge | 重現 → Discovery／規劃 → 實作 → 審核 |

不得因「很小」而略過風險判定；任何行為、契約、資料或安全影響皆不得列 T1。T2 範圍擴大或發現未知根因時，立刻升級 T3；紅線依 canonical §5 為 T4。
